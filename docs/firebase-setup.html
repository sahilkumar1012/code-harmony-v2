🔥 Firebase Setup & Deployment

Step-by-step guide to configure Firebase Authentication, Firestore, and Hosting for Code Harmony v2.

Prerequisites

  • Node.js ≥ 18 and npm ≥ 9
  • A Google account with access to Firebase Console
  • The project cloned locally: git clone <repo-url>

Setup Steps

Create a Firebase Project

Go to Firebase Console and create a new project (or use the existing codeharmonyapp project). Disable Google Analytics if not needed — it can be added later.

Enable Authentication → Google Provider

Navigate to Build → Authentication → Sign-in method and enable the Google provider. Set a public-facing project name and support email.

💡 Note: The app currently uses Google Sign-In only. Additional providers (GitHub, email/password) can be enabled later without code changes — AuthContext.jsx already wraps the Firebase SDK.

Enable Firestore Database

Go to Build → Firestore Database and click Create database. Choose production mode (or test mode for development). Select a region close to your users (e.g., us-central1).

Install Firebase CLI

npm install -g firebase-tools

Verify the installation:

firebase --version

Login to Firebase

firebase login

This opens a browser window for Google authentication. Once authenticated, the CLI is linked to your Firebase account.

Create the .env File

In the project root, create a .env file with your Firebase configuration values. You can find these in Firebase Console → Project Settings → General → Your apps → Web app → Config. 

VITE_FIREBASE_API_KEY=AIzaSy...
VITE_FIREBASE_AUTH_DOMAIN=codeharmonyapp.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=codeharmonyapp
VITE_FIREBASE_STORAGE_BUCKET=codeharmonyapp.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=123456789
VITE_FIREBASE_APP_ID=1:123456789:web:abcdef
⚠️ Important: Never commit .env to version control. Ensure it is listed in .gitignore.

Build the Project

npm run build

Vite compiles the app into optimised static files in the dist/ directory.

Deploy to Firebase Hosting

firebase deploy

This uploads dist/ to Firebase Hosting. After deployment, your site is live at https://codeharmonyapp.web.app.

✅ Done! Your site is now live with global CDN, free SSL, and automatic cache invalidation.

Custom Domain Setup (Optional)

In Firebase Console → Hosting → Add custom domain, enter your domain (e.g., codeharmony.in). Firebase provides DNS records (A / CNAME) to add at your domain registrar. SSL is provisioned automatically.

Environment Variables Reference

VariableDescriptionWhere to Find
VITE_FIREBASE_API_KEY Public API key for Firebase SDK Project Settings → General
VITE_FIREBASE_AUTH_DOMAIN Auth domain for Google Sign-In popup Project Settings → General
VITE_FIREBASE_PROJECT_ID Unique project identifier Project Settings → General
VITE_FIREBASE_STORAGE_BUCKET Cloud Storage bucket URL Project Settings → General
VITE_FIREBASE_MESSAGING_SENDER_ID Cloud Messaging sender ID Project Settings → Cloud Messaging
VITE_FIREBASE_APP_ID Firebase app identifier Project Settings → General

Local Development

Start the Vite dev server with hot module replacement:

npm install      # install dependencies (first time only)
npm run dev       # starts dev server at http://localhost:5173

The dev server reads .env automatically. Any file change triggers instant HMR in the browser — no full reload required.

💡 Tip: Use firebase emulators:start to run Auth and Firestore locally without touching production data.

firebase.json Configuration

Ensure your firebase.json includes SPA rewrite rules for React Router:

{
  "hosting": {
    "public": "dist",
    "ignore": ["firebase.json", "**/.*", "**/node_modules/**"],
    "rewrites": [
      {
        "source": "**",
        "destination": "/index.html"
      }
    ]
  }
}

CI/CD with GitHub Actions (Optional)

Automate deployments by adding a GitHub Actions workflow. Firebase provides a built-in setup command: 

firebase init hosting:github

This creates two workflow files in .github/workflows/:

  • firebase-hosting-pull-request.yml — deploys a preview on every PR
  • firebase-hosting-merge.yml — deploys to production on merge to main
🔑 Secret required: The workflow needs a FIREBASE_SERVICE_ACCOUNT_CODEHARMONYAPP secret in your GitHub repo settings. The firebase init command can create and configure this automatically.

Manual Workflow Example

name: Deploy to Firebase Hosting
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm run build
      - uses: FirebaseExtended/action-hosting-deploy@v0
        with:
          repoToken: ${{ secrets.GITHUB_TOKEN }}
          firebaseServiceAccount: ${{ secrets.FIREBASE_SERVICE_ACCOUNT_CODEHARMONYAPP }}
          channelId: live
          projectId: codeharmonyapp
